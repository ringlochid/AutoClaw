# Release guide

Status: Reference

Use this page when the question is about what we publish and which install paths we support.

## Key rules

- publish from the root packaging surface
- keep the release story aligned to PyPI artifacts
- keep `pipx` as the default public lane
- keep `uv` as a supported secondary lane over the same artifacts
- do not teach convenience installers as shipped v1 support until they are tested and documented
