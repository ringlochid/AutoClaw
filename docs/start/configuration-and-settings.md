# Configuration and settings

AutoClaw keeps local settings in `config.toml`, but the command output is the easiest way to inspect a live install.

## Find the config file

```bash
autoclaw config path
autoclaw config path --json
```

Most commands also accept an explicit config path:

```bash
autoclaw config show --config /path/to/config.toml --json
```

Use `--config` in scripts when you run more than one AutoClaw install on the same machine. For shells or service units that cannot pass flags, `AUTOCLAW_CONFIG=/path/to/config.toml` selects the same file.

## Check effective settings

```bash
autoclaw config show --json
```

The output redacts local secrets. Key fields:

- `server.host` and `server.port`: AutoClaw API, MCP, and packaged console bind target
- `paths.data_dir`: local data directory
- `database.url`: controller database
- `openclaw.base_url`: local OpenClaw Gateway URL
- `openclaw.agent_id` and `openclaw.operator_agent_id`: worker and operator agents

Open the console at:

```text
http://127.0.0.1:<server.port>/
```

The default port is `18125`. First task launch lives at `/task-start`; task detail pages live at `/tasks/<task_id>`.

## Data directory

`paths.data_dir` contains local controller data, wrapper material, definition drafts, per-task directories, and the default SQLite database.

Common default examples:

- Linux: `~/.local/share/autoclaw`
- Windows: `%LOCALAPPDATA%\autoclaw`

Per-task directories live under:

```text
<data_dir>/tasks/<task_id>/
```

The first-run topic research workflow writes `workspace/research_brief.md` in the task directory and publishes it as the `research_brief` artifact.

The full task-owned workspace path is:

```text
<data_dir>/tasks/<task_id>/workspace/research_brief.md
```

## Annotated config example

Generated config files are comment-free, and AutoClaw rewrites can remove comments. This example explains the fields; do not rely on comments staying in your local file.

```toml
[paths]
# Local controller data. Per-task directories live under <data_dir>/tasks/.
data_dir = "/home/you/.local/share/autoclaw"

[database]
# Default SQLite database. Use postgresql+asyncpg://... with autoclaw[postgres].
url = "sqlite+aiosqlite:////home/you/.local/share/autoclaw/autoclaw.persistence"
echo = false

[server]
# Local API, MCP, and packaged console bind target.
host = "127.0.0.1"
port = 18125

# Development console origins. The packaged console uses the same origin.
console_origins = [
  "http://127.0.0.1:5173",
  "http://localhost:5173",
  "http://127.0.0.1:4173",
  "http://localhost:4173",
]

[logging]
level = "WARNING"

[security]
# Generated by onboarding or init. Do not paste this value into issues or chat.
api_key = "<generated secret>"

[openclaw]
# OpenClaw Gateway. Loopback is the supported v1 shape.
base_url = "http://127.0.0.1:18789"

# Dedicated OpenClaw agents created or reconciled by AutoClaw setup.
agent_id = "autoclaw-worker"
operator_agent_id = "autoclaw-operator"
timeout_ms = 120000

[runtime]
# Provider-start retry and signal-driven watchdog defaults.
dispatch_launch_retry_initial_backoff_seconds = 1.0
dispatch_launch_retry_max_backoff_seconds = 30.0
watchdog_inactivity_timeout_seconds = 900
watchdog_same_attempt_replacement_limit = 2
```

## After changes

Restart the running foreground `autoclaw serve` process or the managed service after changing bind, database, OpenClaw, or runtime settings.

Then check:

```bash
autoclaw doctor
autoclaw openclaw check
autoclaw config show --json
```
