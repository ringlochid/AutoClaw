# Concepts

AutoClaw concepts are easiest to learn in layers. Start with why orchestration exists, then learn authoring nouns, then runtime nouns.

## Learning path

1. [Orchestration model](orchestration-model.md) - general concept teaching: orchestration, harnesses, OpenClaw, MCP, A2A, and why the controller/files/prompt layers exist.
2. [Core concepts](core-concepts.md) - the compact vocabulary: workflow, task-compose, assignment, checkpoint, artifact.
3. [Authoring model](authoring-model.md) - reusable definition teaching: role, policy, workflow, node, criteria, consumes, produces.
4. [Runtime model](runtime-model.md) - runtime teaching: task, flow, manifest, assignment, attempt, dispatch, checkpoint, artifact, boundary, wait, replan.
5. [Capability model](capability-model.md) - human requests, command runs, budgets, retry, and replan.
