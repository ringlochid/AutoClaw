# Concepts

AutoClaw concepts are organized by layer: orchestration purpose, reusable authoring nouns, and runtime nouns for one launched task.

## Learning path

1. [Product overview](overview.md) - the shortest orientation to the shipped product and its main surfaces.
2. [Orchestration model](orchestration-model.md) - general concept teaching: orchestration, harnesses, OpenClaw, MCP, A2A, and why the controller/files/prompt layers exist.
3. [Core concepts](core-concepts.md) - the compact vocabulary: workflow, task-compose, assignment, checkpoint, artifact.
4. [Authoring model](authoring-model.md) - reusable definition teaching: role, policy, workflow, node, criteria, consumes, produces.
5. [Runtime model](runtime-model.md) - runtime teaching: task, flow, manifest, assignment, attempt, dispatch, checkpoint, artifact, boundary, wait, replan.
6. [Capability model](capability-model.md) - human requests, command runs, budgets, retry, and replan.
