# Handle ambiguity and incidents

Workers should name uncertainty and stay inside their assignment. Parents and roots should route it.

Use this order for an incident:

1. stop or contain harmful work
2. preserve controller state and evidence
3. identify the affected scope
4. assign focused investigation
5. choose retry, replan, human request, or blocked closure
6. verify recovery
7. record residual risk

Do not turn surprising behavior into an automatic retry. If the workflow is solving the wrong problem, replan or ask for direction instead of widening a worker's scope.

Use human requests for judgment and command runs for long command work. These are separate capabilities.
