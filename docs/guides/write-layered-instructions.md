# Write layered instructions

Put each sentence in the narrowest reusable layer where it stays true.

| Layer | Put this here | Keep this out |
| --- | --- | --- |
| Role instruction | reusable specialist behavior | one task's paths and scope |
| Policy instruction | authority and capability rules | job identity |
| Node instruction | local mission and scope | copied role or policy text |
| Task instruction | details for this launch | reusable doctrine |
| Criteria | hard pass or fail rule | preferences |
| Produces | durable outputs | status messages |
| Consumes | named required evidence | vague context reminders |

Keep `description` short and scannable. Put procedures in `instruction`.

Before publishing, remove repeated wording. If many nodes need the sentence, move it to a role or policy. If it applies only to this run, move it to task-compose.

See [write a role](write-a-role.md), [write a policy](write-a-policy.md), and [write a workflow](write-a-workflow.md).
